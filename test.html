Setting up NJE on VSE

Option 2
 This guide is going to cover setting up NJE networking on VSE.
It's relatively simple to do, but, there are a few gotchas that you might encounter.

The first thing we need to do is regenerate the POWER load phase to allow for PNET networking to exist. To do this, go to ICCF library 59 and copy SKPWRNDT. Change the phase name to PNETNDT, and define any nodes you may want to have. Example:
===> *MSG=PRESS PF1 TO SHOW PF KEY ASSIGNMENT ...+....5....+.. MEM=PWRNDT >>..+..FS * $$ JOB JNM=IESPNDT,CLASS=0,DISP=D /===/* * $$ LST CLASS=Q *===* // JOB POWER NETWORK DEFINITION TABLE *===* // LIBDEF PHASE,CATALOG=PRD2.CONFIG *===* // OPTION CATAL *===* // EXEC ASMA90,SIZE=(ASMA90,64K),PARM='EXIT(LIBEXIT(EDECKXIT)),SIZE(MAXC *===* -200K,ABOVE)' *===* PRINT NOGEN *===* PNETNDT PNODE NODE=VSEOLD,APPLID=VSEOLDPN,LOCAL=YES *===* PNODE NODE=VMESA,LOCAL=NO,AUTH=JOB *===* END *===* /* *===* // EXEC LNKEDT *===* /& *===* * $$ EOJ *===* ***** END OF FILE ***** ***** Now that you've done that, submit that job and invoke DITTO. Copy PNETNDT.PHASE from PRD2.CONFIG to IJSYSRS.SYSLIB, or change the CATALOG line in the job. Do note that this is somewhat risky (cataloguing directly into the system library)! Next, copy the SKPWRGEN sample from ICCF library 59, and, edit to to tell it that PNET works and exists, like so:
===> *MSG=PRESS PF1 TO SHOW PF KEY ASSIGNMENT ...+....5....+.. MEM=PWRGEN >>..+..FS * $$ JOB JNM=POWERGEN,CLASS=0,DISP=D /===/* * $$ LST CLASS=Q *===* // JOB POWER GENERATION *===* LIBDEF PHASE,CATALOG=PRD2.CONFIG *===* // OPTION CATAL *===* // EXEC ASMA90,SIZE=(ASMA90,64K),PARM='EXIT(LIBEXIT(EDECKXIT)),SIZE(MAXC *===* -200K,ABOVE)' *===* PWR TITLE 'VSE/POWER - IPWPOWER GENERATION ' *===* EJECT *===* PRINT NOGEN *===* SPACE 3 *===* IPWPOWER POWER * *===* ACCOUNT=YES, * *===* CLRPRT=YES, * *===* COPYSEP=YES, * *===* DBLKGP=10, * *===* DBLK=0, * *===* FEED=NO, * *===* JLOG=YES, * *===* JSEP=(0,0), * /===/* LTAB=(10,00,05,10,15,20,25,30,35,40,45,50,56), * *===* MEMTYPE=P, * *===* MRKFRM=YES, * *===* MULT12=NO, * *===* NTFYMSG=100, * *===* PAUSE=NO, * *===* PNET=PNETNDT, * *===* PRI=3, * *===* RBS=(0,0), * *===* SECNODE=AAAA, * *===* SHARED=NO, * *===* STDCARD=(0,0), * *===* STDLINE=(0,0), * *===* SPLIM=90, * *===* SPOOL=YES *===* EJECT *===* */INCLUDE SKPWRBSC *===* */INCLUDE SKPWRSNA *===* END *===* /* *===* // EXEC LNKEDT,PARM='MSHP' *===* /& /===/* * $$ EOJ *===* ***** END OF FILE ***** ***** Once you've done this, submit the job and re-invoke DITTO (or use LIBR) to copy the IPWPOWER.PHASE load module into IJSYSRS.SYSLIB from PRD2.CONFIG. Now, go to the Hardware Configuration and IPL screen and enter E20 as a CTCA. Next, re-IPL. Once you have loaded the system back up, enter this to start networking:
S PNET,VMESA,,E20 This will instruct POWER to attempt to start a NJE connection to the node named VMESA over the line with the address E20. Since this is a VM system that this example VSE system is running under, you may want to enter these RSCS commands to start NJE on that end (run these as RSCS)
CP DEFINE CTCA 20 CP COUPLE 20 VSE E20 RSCS DEFINE VSEOLD TYPE NJE LINE 20 NODE VSEOLD RSCS START VSEOLD Once you've done that, you should see the sign-on message, and, everything should be functional.
